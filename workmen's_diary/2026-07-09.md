### 2. อธิบาย/วางแผนการตั้งค่า Label Studio 3 โปรเจกต์
อธิบายขั้นตอนสร้างโปรเจกต์ (Create Project → Labeling Setup → Code → paste XML → Data Import → upload JSON) — ย้ำว่าต้องตั้ง Labeling Setup **ก่อน** Data Import เสมอ (ไม่งั้น error "more than one data key")

- Label Studio `visibleWhen` รองรับแค่เงื่อนไขจาก action ของ annotator (`choice-selected` ฯลฯ) ไม่รองรับเงื่อนไขจากข้อมูลดิบของ task
- มี bug ยืนยันจากทีม Label Studio เอง (**GitHub Issue #3313**) ว่า `visibleWhen` ทำงานผิดพลาด โชว์/ซ่อนแบบไม่สนใจค่าเจาะจงที่เลือกจริง

สรุปแนะนำมะขาม: **แยก 3 โปรเจกต์ตามเดิม** ปลอดภัยกว่า เร็วกว่า ไม่มีความเสี่ยงข้อมูลแสดงผิด

### 4. มะขามเริ่มสร้างโปรเจกต์จริงใน Label Studio Cloud — เจอบั๊ก scroll จริง
มะขามสร้างโปรเจกต์ Elements สำเร็จ (project 271482) แล้วส่ง screenshot มาว่า **"tab ซ้ายเลื่อนขึ้นลงไม่ได้แล้วจะดูแปลนยังไง"** — วินิจฉัยพบว่า `.stickycol` (คอลัมน์ปักรูป+header) ไม่มี `max-height`/`overflow-y:auto` พอเนื้อหาสูงกว่าจอ ส่วนเกินโดนตัดหายไปเฉยๆ มองไม่เห็นเลย ไม่ใช่แค่ "เลื่อนไม่ได้" แต่ "เนื้อหาหายไปจริง"

**แก้:** เพิ่ม `max-height:96vh; overflow-y:auto; padding-right:6px;` ให้ `.stickycol` ใน **ทั้ง 5 ไฟล์** (elements/elements-freezeR/single/single-freezeR/material_list)

### 5. ลบ/ถอน browser "no style information" fix เพราะทำให้แย่ลง
ระหว่างแก้ข้อ 4 มะขามขอให้เอาข้อความเตือน "This XML file does not appear to have any style information..." ออกด้วย (ข้อความมาตรฐานของ browser ตอนเปิดไฟล์ `.xml` ตรงๆ) — เพิ่ม `<?xml-stylesheet type="text/css" href="data:text/css,*{}"?>` ไปบนสุดทั้ง 5 ไฟล์

**ผลข้างเคียงที่ไม่คาดคิด:** พอมะขามเปิดไฟล์ตรงๆ ใน tab ใหม่ กลายเป็นโชว์ข้อความมั่วๆ (CSS ของ `<Style>` และข้อความใน `<Choice>` หลุดออกมาปนกัน ไม่มีโครงสร้าง) เพราะ browser เปลี่ยนโหมด render พอเจอ stylesheet (แม้จะว่างเปล่า) — มะขามงงและหงุดหงิด ("อะไรวะ") → อธิบายสาเหตุแล้ว **ถอนการแก้ทั้งหมดออก** (ลบบรรทัด `<?xml-stylesheet?>` ออกจากทั้ง 5 ไฟล์) และย้ำว่าไฟล์พวกนี้ไม่ได้ถูกออกแบบให้เปิดตรงๆ ใน browser เลย ต้อง paste เข้า Label Studio Code editor เท่านั้นถึงจะเห็นผลถูกต้อง

### 6. ปรับ Material List ให้รูปปักซ้ายเหมือน Elements
`label-studio-makham-material_list.xml` เดิมใช้ layout รูปแปะด้านบนแบบ sticky-top (ของเก่าตั้งแต่ก่อนยุค Makham) — เปลี่ยนเป็น sticky-left (รูปปักซ้าย คอลัมน์ข้อมูลเลื่อนขวา) ให้ตรงกับ elements.xml เพื่อความสม่ำเสมอทั้ง 3 โปรเจกต์

### 7. ย้าย header fields ออกจากคอลัมน์รูป ไปไว้ฝั่งข้อมูลแทน
มะขามขอ **"ให้ซ้ายมีแต่รูปภาพ"** — ย้าย header ทั้งหมด (บ้าน/หน้า/Sheet/ชื่อแผ่น/Pattern/View) ออกจาก `.stickycol` ไปไว้บนสุดของคอลัมน์ข้อมูลฝั่งตรงข้ามแทน ทำใน**ทั้ง 5 ไฟล์** — คอลัมน์รูปตอนนี้มีแค่ `<Image>` อย่างเดียว ขยาย `maxHeight` เป็น `94vh` เต็มที่

### 8. ปรับ resize คอลัมน์ — ไม่มี drag-resize จริง เปลี่ยนเป็น 50/50 แทน
มะขามถามว่าปรับขนาดคอลัมน์ซ้ายด้วยการลากได้ไหม — เช็คแล้ว **Label Studio ไม่มี splitter tag ให้ผู้ใช้ลากปรับ** ในหน้า config ที่สร้างเอง (ฟีเจอร์ resize ที่มีจริงเป็นของแผง Outliner/Details ในตัว Label Studio เอง คนละส่วนกับ View ที่เราออกแบบ) → ทำ **50/50 คงที่แทน** ตามที่มะขามเสนอเป็นทางเลือกสำรอง (`flex:1 1 0%; max-width:50%` ทั้ง 2 ฝั่ง) ใน**ทั้ง 5 ไฟล์**

### 9. Redesign header block ของ Single ให้ไม่รกเปลืองที่ (Elements/Material List ไม่แตะ)
มะขามส่ง screenshot โปรเจกต์ Single บ่นว่า **"ทำไมมันทุเรศจังวะ"** — header 5 ช่อง (บ้าน/หน้า/Sheet/ชื่อแผ่น/Pattern) แต่ละช่องซ้อนกันเป็นแถวเต็มความกว้าง มีช่องว่างเปล่าๆ มหาศาลระหว่างแถวก่อนจะถึงเนื้อหา JSON จริง

**แก้:** ออกแบบ `.hdrbar`/`.hdrfield` ใหม่ — จัด header ทั้ง 5 ช่องเป็นแถบแนวนอนเดียว (flex-wrap) แทนการซ้อน 5 แถวแนวตั้ง ลดพื้นที่ว่างไปเกือบหมด — **มะขามระบุชัดว่าทำแค่ `single.xml`/`single-freezeR.xml` เท่านั้น ไม่เอาไป elements/material_list** (สั่งยกเลิกกลางคันตอนกำลังจะแก้ elements ด้วย) — เคารพตามนั้น elements/material_list ยังเป็น layout แบบเดิม (ซ้อนแนวตั้ง) อยู่
