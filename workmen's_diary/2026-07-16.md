## [ช่วงเช้า] มะขาม (ทำงานร่วมกับ Claude Code — ต่อเนื่องจากรีวิวบ้านหลังที่ 2, ดู `2026-07-15.md`)

**บริบท:** ต่อจากที่ทำ house 02 review เมื่อวาน มะขามสั่งงานใหม่ 2 อย่าง: (1) เพิ่ม dummy grid ใหม่ 6 เส้นในกริดมาสเตอร์ของบ้าน 2 ด้วยค่าที่มะขามให้ตรงๆ พร้อมกำชับ "อย่าคิดแทนผมมาก" (2) ให้ cascade แก้ทุกหน้าที่เกี่ยวข้องกับ หน้า00 โดยเฉพาะ pattern=plan (3) แปลไฟล์ `สิ่งที่ต้องแก้.md` ทั้งหมดเป็นภาษาอังกฤษเพื่อประหยัด token

**งานที่ทำ:**

### 1. เพิ่ม dummy grid 6 เส้นในกริดมาสเตอร์บ้าน 2 (`หน้า00_gridline.json`)
มะขามให้ค่ามาตรงๆ: x-axis `1'`=-1.0, `1''`=1.8 (จากหน้า12), `3'`=7.7, `4'`=10.6 (จากหน้า06); y-axis `F'`=-0.6, `F''`=0.9 (จากหน้า06) — เพิ่มเป็น dummy entries, confidence_score:1, ไม่ใส่ reasoning เพิ่มเติมเกินจำเป็นตามที่สั่ง

### 2. Cascade แก้หน้าที่เกี่ยวข้อง (เช็คกับภาพก่อนแก้ทุกจุด)
- **หน้า06_floor_plan.json**: ซูมดู dimension chain แถว F พบว่า "0.60"+"0.90" ที่เคยเข้าใจว่าเป็น overhang ต่อเนื่อง 1.50m จริงๆ แล้วเป็นคนละจุดกัน (0.60=ภายนอก, 0.90=พาร์ทิชันภายใน) แชร์จุดเดียวกันที่ gridline F พอดี — แก้ ซักล้าง จาก "~1.50m strip" เป็น "F-F'' x 1-2" (แม่นยำ), ห้องครัว เป็น "F''-E x 1-2", ห้องนอน1 ระบุจุด sub-split ว่าคือ 3' (6.4+1.30=7.7 ตรงกันเป๊ะ), ห้องน้ำ1 (floor-plan box) เป็น "E-D x 1'-1''" — เขียน warning paragraph ใหม่ทั้งหมด (ของเดิมสรุปผิดว่า "ไม่ต้องมี dummy grid" ซึ่งตอนนี้ไม่จริงแล้วตามคำสั่งมะขาม)
- **หน้า12_wc1_plan.json**: ห้องน้ำ WC1 หลัก จาก "~1" เป็น "1'-1''" — width breakdown เดิม [1.00,1.80] รวมกันได้ 2.80 ตรงกับ 1'-ถึง-grid1(1.00) และ grid1-ถึง-1''(1.80) เกือบเป๊ะ น่าจะเป็นที่มาโดยตรงของ dummy grid 2 เส้นนี้
- **หน้า39_floor_plan.json**: grease_trap จาก "F-E x ~1" เป็น "F''-E x ~1" (คำอธิบายเดิมบอกอยู่แล้วว่า "adjacent to ห้องครัว")

### 3. พบปัญหาเดิมที่ไม่เกี่ยวกับรอบนี้ — ไม่ได้แก้ ใส่ flag ไว้เฉยๆ
ระหว่างแก้หน้า12 พบว่า `dimension_m` เดิมอ้างว่า "2.70 (E-D span)" แต่กริดมาสเตอร์จริงบอก E-D=2.0m (E=4.0,D=6.0) ไม่ตรงกัน — ไม่ใช่เรื่องที่มะขามสั่งแก้รอบนี้ ใส่ flag ไว้ให้คนตรวจสอบ ไม่ได้ตีความเองแล้วแก้ทันที

### 4. จุดที่ตัดสินใจ "ไม่ cascade" เพราะหลักฐานไม่พอ
- ST2 (หน้า06) "F x ~1" — อาจเกี่ยวกับ F'/1' แต่ไม่มี dimension ชัดเจนที่ผูกได้แน่นอน
- Septic tank ตัวแรก (หน้า39) "F-E x ~1" — คำอธิบายกำกวมระหว่างโซน F-F'' กับ F''-E
- หน้า07_roof_plan.json overhang ตะวันตก/ตะวันออก (1.30m) — เช็คแล้วไม่ตรงกับ 1'(-1.0)/4'(+1.0) เพราะเป็นคนละระยะ (ชายคาหลังคายื่นไกลกว่าขอบผนัง/กันสาดที่ dummy grid ใหม่นี้อ้างถึง) — ไม่แตะ

### 5. แปล `สิ่งที่ต้องแก้.md` ทั้งไฟล์เป็นภาษาอังกฤษ
มะขามสั่งให้แปลเนื้อหาทั้งหมดเป็นอังกฤษเพื่อประหยัด token ตอน AI อ่านไฟล์นี้ในอนาคต — แปลทั้ง 22 ข้อ (บ้าน 1 + บ้าน 2) คงโครงสร้าง/หมายเลขข้อ/path/JSON snippet ไว้เหมือนเดิมทุกอย่าง เก็บชื่อห้อง/element_id ภาษาไทยไว้ตามเดิม (เพราะเป็นค่าจริงในข้อมูล JSON ไม่ใช่ narrative) แปลเฉพาะส่วนอธิบาย/เหตุผล

**ไฟล์ที่เปลี่ยนวันนี้:** `หน้า00_gridline.json`, `หน้า06_floor_plan.json`, `หน้า12_wc1_plan.json`, `หน้า39_floor_plan.json`, `สิ่งที่ต้องแก้.md` (แปลทั้งไฟล์ + เพิ่มข้อ 23)
