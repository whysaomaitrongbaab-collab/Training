- รอบแรก: สร้าง `primary_rawjson_schema.json` (ย่อจาก `20260708draft of prime rawjson.md` 45KB เหลือ 10KB) — มะขามบอกว่า **"มันดูสั้นจัง"** ไม่พอ ต้องการเป็นไฟล์ `.md` แทน
- รอบสอง: มะขามชี้แจงชัดขึ้น — **อย่าลบต้นฉบับ**, ใส่เนื้อหาให้ครบ (ไม่ใช่ย่อโหด), ย่อแค่บางส่วนเพื่อประหยัด token, พิจารณาเขียนอังกฤษถ้าช่วยประหยัด token จริง → เขียนใหม่เป็น `primary_rawjson_schema.md` (11.8KB, ~26% ของต้นฉบับ) เก็บกติกาปฏิบัติทุกข้อครบ ตัดแค่ประวัติ/เหตุผลการตัดสินใจ Gen2-vs-Makham ออก
- ลบไฟล์ `.json` ตัวแรกทิ้งตามคำสั่งมะขามหลังยืนยันว่าจะใช้ `.md` ต่อ

### 2. ตั้งโครงสร้างใหม่ `raw_json_ตัวที่ใช้งานจริง/` (repo root)
มะขามสร้างโฟลเดอร์นี้ไว้แล้วเอง (`00file_for_making_rawjson_from_claude/` — ตอนแรกสะกดไม่มี `00` prefix มะขามสั่งเปลี่ยนชื่อให้มี `00` นำหน้าระหว่างทำ) — ย้าย `primary_rawjson_schema.md` เข้าไปที่นั่น แล้ว **copy** (ไม่ใช่ move) `training-data/mk_test/t4/` → `raw_json_ตัวที่ใช้งานจริง/01บ้าน_เล็ก_1ชั้น_01/` (101 ไฟล์ครบ) เป็นจุดเริ่มของ "บ้านหลังที่ 1 ที่ใช้งานจริง"

เขียน `README.md` อธิบาย workflow เต็ม (โครงสร้างโฟลเดอร์ + กติกาตั้งชื่อ `0N<house>` + ขั้นตอนถอดแบบ + ขั้นตอนเตรียมข้อมูลเข้า Label Studio) — ตอนแรกมีคำเตือนว่า generator script ยังผูกกับ `mk_test/` เท่านั้น ยังไม่รู้จักโฟลเดอร์นี้

### 3. ผูก `label-studio-tasks-makham.js` ให้อ่านจาก `raw_json_ตัวที่ใช้งานจริง/` อัตโนมัติ

### 5. คำถามจริง: ภาษาไทยกับอังกฤษ อันไหนประหยัด token กว่า
มะขามถามตรงๆ — ตอบว่า **English ประหยัดกว่า 2-4 เท่าโดยทั่วไป** (tokenizer ฝึกด้วยข้อมูลอังกฤษเป็นหลัก มี subword merge ให้คำอังกฤษเยอะกว่ามาก ภาษาไทยมักถูกตัดเป็นชิ้นเล็กระดับพยางค์) พร้อมข้อแม้ว่า field name ใน JSON เป็นอังกฤษอยู่แล้ว ส่วนที่ได้ประโยชน์จริงคือ prose/คำอธิบาย

### 6. แปล `primary_rawjson_schema.md` + `README.md` เป็นอังกฤษเต็ม
